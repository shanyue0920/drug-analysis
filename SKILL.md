# 药品分析 Skill
> 作者 / 出品：周维

对给定药品/治疗领域进行结构化综合分析，将临床档案与商业评估合并为一份报告，并可生成患者科普素材。

## 1. 概述
回答"这个产品是什么、临床凭什么、市场有多大、能不能赢、风险在哪、怎么讲给患者和医生听"——
一份报告同时给到临床档案与商业判断，适合产品立项、全面评估、向上汇报、内容创作。

## 2. 前置检查
- 未给出明确药品名称/治疗领域时，先追问，绝不凭空分析。
- 时效词（最新/当前/市场规模/集采/医保/在研/获批）必须 WebSearch 联网核实，不凭记忆。
- 市场/患者数据必须标来源与口径，区分已验证值与估算值。

## 3. 执行步骤
1. 产品概览：提取基本信息（通用名/商品名/企业/剂型/规格/批准文号/药理分类），凝练一句话定位（临床价值主张 ∩ 商业定位）。
2. 作用机制与靶点：通路、靶点、药理分类（见 `references/drug-classes.md`）。
3. 适应症与循证证据：按 `references/evidence-levels.md` §4 **强制罗列循证三件套**——①研究数据（关键 RCT/Meta/RWE，每条含研究名·设计·样本量·关键终点·年份·证据等级）②指南推荐（指南名·机构·年份·推荐级别·具体地位）③专家共识（共识名·机构·年份·核心建议）；任一缺失标 💭 待核实，禁止用单源/厂商材料替代独立循证。
4. 用法用量与特殊人群：剂量、途径、儿童/孕妇/肝肾注意。
5. 药代动力学(PK)：ADME、半衰期、达峰时间。
6. 安全性：不良反应/禁忌/相互作用/黑框警示。
7. 注册审批与在研管线：NMPA/FDA 状态、专利、在研。
8. 疾病市场与流行病学：患病人数、发病率趋势、未满足需求。
9. 患者流漏斗：就诊→确诊→治疗→达标，定位流失瓶颈（见 `references/market-methodology.md`）。
10. 竞争格局：直接/间接/替代竞品对比。
11. 政策准入与医保集采：医保类别、集采影响、准入路径。
12. 支付方与药物经济学：支付结构、成本效果/ICER。
13. 渠道与终端：医院/零售/DTP/线上占比、进院节点。
14. 处方者观念与患者旅程：处方驱动/障碍、患者依从路径。
15. 综合定位与策略：SWOT + 差异化定位主张 + **关键市场机会与风险（按 `references/market-methodology.md` §5 方法产出：机会须含「量级 + 杠杆点」、风险须含「落地抓手」，禁止只写方向性口号）**。
16. 科普素材（如要求）：基于事实生成口语化科普，带免责声明。
17. 全部数据沿用第 6 节校准规则（🔍/💭、交叉校验、时效标注、口径标注）。

## 4. 输出格式

> **模板与落盘**：完整结构见 `templates/combined-report.md`，建议对照输出以保证结构统一。若用户明确要求「生成文件 / 落盘」，使用 `Write` 把报告写入当前工作区的 `outputs/` 目录（不存在则先提示创建），文件名按 `YYYYMMDD_{产品}_综合分析.md` 命名，并完整保留 🔍/💭 标注、截至日期与免责声明；未要求落盘时仅在对话内输出，不擅自写文件。

## 5. 约束
- **结构完整性（强制，详见第 7 节）**：报告必须完整保留全部章节与子节；任一维度查不到内容时须保留空章节并标「待核实（💭）」，禁止删除章节、合并章节或以「无/暂略」跳过，禁止编造数据填充空字段。
- 禁止编造批准文号、临床/销售数据、价格、获批状态；来源不明标「待核实」。
- 市场规模/患者数/份额标来源与口径，估算项标「估算」。
- 集采/医保/审批/在研等时效信息必须联网核实并标截至日期。
- 疗效/安全/获批/SWOT 结论须基于前述事实，不得凭空得出。
- 科普素材基于事实、不夸大、带免责声明；不替代专业医疗或商业尽调。
- 落盘报告须沿用 `templates/combined-report.md` 结构、数据校准标注与免责声明，文件名含日期；未要求落盘时仅在对话内输出，不擅自写文件。

## 6. 数据校准规则（强制）

本 Skill 输出的每条关键数据都必须可溯源、可校验。严格执行以下 5 条：

- **来源分级标注**：每条关键数据标可信度 —— 🔍 已查证（官方说明书 / NMPA·FDA / 权威指南 / 核心文献）、💭 推测或不确定（模型推断或单一非权威来源）。禁止把 💭 当 🔍 呈现。
- **关键事实交叉校验**：批准文号、医保类别、集采状态、价格、获批适应症等，须至少 2 个独立来源一致方可写为确定；不一致或仅单源时标「待核实」。
- **时效校准**：集采/医保/审批/在研状态必须 WebSearch 取最新，并标注「截至 YYYY-MM」，不得沿用旧记忆。
- **口径校准**：市场规模/患者数注明统计口径（终端价/出厂价、中国/全球、确诊/估测）；估算项显式标「估算」。
- **禁止编造**：任何无法溯源的数据标「待核实」，不得生成看似合理但无来源的数字。

## 7. 输出完整性规则（防偷懒·强制）

本 Skill 的输出结构是**写死的**，任何情况下不得为"省事"而改动：

- **章节不可删**：报告必须完整包含「一~五」全部章节与「2.1~4.3」全部子节。某维度查不到任何信息，须**保留空章节并标注「未查到相关内容（💭 待核实）」**；严禁删除该节、并入他节，或用「无」「不适用」「暂略」一笔带过。
- **字段不可跳**：模板每个字段（如研究名、样本量 N=、指南具体地位、医保限定范围）查不到时**留空或写「待核实」**，不得因"懒得查"删除整条，也不得凭印象补一个看似合理的值。
- **禁止编造填充**：空章节/空字段宁可显式留空并标 💭，也**绝不**生成无来源的批准文号、样本量、价格、循证结论来"填满"。
- **不存在 ≠ 可删**：若某类证据（如专家共识、在研管线、某国获批状态）公开检索确无结果，写明「未检索到公开 {X} 信息（💭）」，而非不输出该子节。
- **结构顺序不可调**：子节编号与层级严格按模板，不因内容多寡重排或省略。
- **自检动作（输出前必做）**：逐项核对模板章节是否齐全；凡缺失一节、或某节被以"无信息"理由省略，均视为违反本规则，须补回后再输出。
