# drug-analysis · 药品综合分析 Skill

一个给 AI 用的「药品/治疗领域结构化分析工作手册」。给定任一药品（化学药/生物药/中成药）或治疗领域，它会一次性产出**融合了临床档案与商业评估的合并报告**，并强制做数据校准、循证三件套罗列、带量级与杠杆的市场机会分析。

> 适用场景：医药产品立项、品牌规划、市场准入论证、向上汇报、医学/市场内容创作。

---

## ✨ 核心特性

1. **单模式「综合分析」**：不在 A/B 模式间纠结——一次触发同时产出临床档案 + 商业评估。
2. **循证三件套（强制）**：研究数据（每条含 设计·样本量·关键终点·年份·证据等级）、指南推荐（写明在指南里的具体地位）、专家共识（单独罗列）。药品是严肃医疗，研究底座必须厚。
3. **市场机会带「量级 + 杠杆点」**：拒绝"扩大适应症、提升渗透率"这类口号；要求自下而上算可服务患者池、给可执行的杠杆抓手。
4. **数据校准规则（强制）**：每条关键数据标 🔍 已查证 / 💭 推测；关键事实交叉校验；时效信息联网并标截至日期；口径标注；禁止编造。
5. **可落盘**：用户要求"生成文件"时按 `YYYYMMDD_{产品}_综合分析.md` 写入 `outputs/`；不强制写盘，遵守"不污染源文件夹"。

## 📐 输出报告结构（节选）

```
一、产品概览

```

## 🔄 自动发布（开发者向）

本仓库内置「改完即发布」能力，维护者无需手动操作：

- **自动发布 + 更新**：`.git/hooks/post-commit` 在每次 `git commit` 后，自动把 Skill 发布/覆盖更新到 SkillHub（skillId=97070）。发布 token 从 **macOS 钥匙串**读取（service=`skillhub` / account=`shanyue0920`），不落明文文件。
- **自动备份**：发布成功后自动 `git push origin main` 同步到 GitHub。
- **结果通知**：发布成功/失败通过 **macOS 原生通知**弹出提醒（符合"提醒走 Apple 原生"规范）。
- **版本唯一性**：每次发布版本号 = `SKILL.md` 的 `version` + `+YYYYMMDDHHMMSS`（精确到秒），避免 SkillHub 版本冲突。

**手动发布兜底**：`bash outputs/publish-skill.sh`（同样从钥匙串读 token，自动升版本）。

> 注意：`.git/hooks/` 是本地 git 元数据，不会被提交/推送；GitHub Actions 另有 workflow，但本账号 runner 受限，自动发布以本地 hook 为准。
